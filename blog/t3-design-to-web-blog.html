Responsive Design

What a responsive site is, and why responsiveness is important.

A reposnive site is deisnged to adapt to different screen sizes and orientations. The elements on a webpage will change their layout depending on the size of the screen. For example a menu bar that is horizontal on a large desktop screen may change to a vertiacal bar when viewed on a mbile phone. Responsioves is important becuase there are so many differente devices to view websites on. A web developer cannot assume that a website will only be displayed on a large screen.

What mobile first design is, and why it's important.

Mobile first design is focusing on the layout of a website on a small screen before considering the layout on larger screens. Because mobile devices are so popular it is more likely your site will be viewed on a smaller screen. Therefore a mobile design should be the priority when designing a website. Larger screens can be catered for but the mobile screen layout should be considered first.

What frameworks are, and their pros and cons.

Frameworks are a structured collection of files (HTML, CSS, JavaScript etc) that can be used to support the development of websites. Frameworks provide a common structure so that developers don't have to re-create a website from scratch. They can use the code that is already provided in the framework.

Framework Pros:

  • Browser compatibility

  • Learn good practices

  • Speeds up the mock-up process

Framework Cons:

  • Unused code leftover

  • Slower learning curve

  • You don't learn to do it yourself

What a wireframe is and why we use it.

Wire framing is the initial stage of website design process. The purpose is to work in a minimal way to start organising content and the flow of information on a page. Wire framing allows you to work quickly and generate lots of ideas with out worrying about styling.
